# Как начать принимать сообщения на ценник по сети LoRa

В текущей реализации существует "шлюз", который по сети LoRa отправляет на ценники новые данные, которые должны быть отображены
Собственно, это небольшая инструкция по запуску этого "шлюза"

"Шлюз" должен быть создан согласно схеме и подключен к пк или иметь элементы питания.
Важно, что "шлюз" должен быть подключен к той же сети WiFi, к которой подключено устройство (пк), где развернут веб интерфейс.

После подключения выполняется для обнаружения имени порта выполняется команда:
```
[System.IO.Ports.SerialPort]::getportnames()
```
P.S.: команда актуальна для powershell (windows). В другой ОС команда может отличаться


После необходимо загрузить на микроконтроллер файл:
```
lora_sender_display.py
```

Это можно сделать командой:
```
py -3.10 -m mpremote connect com10 cp .\to_controller\lora_sender_display.py :main.py
```
com10 - может отличаться, узнается командой "...getportnames"

Далее перезагружаем контроллер и запускаем
```
py -3.10 -m mpremote connect com10 reset
py -3.10 -m mpremote connect com10 ls
```

Должны увидеть вывод 
```
......
[WIFI] Connected!
[WIFI] IP: 10.133.210.157
[WIFI] Device: TAG-101
[SERVER] Starting on 10.133.210.157:80
[SERVER] Ready
[SERVER] API: http://10.133.210.157/api/price
[SERVER] LoRa: ON

```

Если этого нет, нажмите Enter, вывод должен появиться.



Теперь устройство готово к приему сообщений от веб интерфейса и отправить их далее по сети LoRa.
